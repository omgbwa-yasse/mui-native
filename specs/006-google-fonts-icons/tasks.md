## Phase 3: User Story 1 — Named Material Icon Component (Priority: P1) 🎯 MVP

**Goal**: A developer can render any Material Icon via `<MaterialIcon name="home" />` with correct theme color and all 5 variant options.

**Independent Test**: Import `MaterialIcon` from `src/components/MaterialIcon/index.ts`, render inside `ThemeProvider`, assert element renders with default size 24.

### Implementation

- [X] T008 [US1] Write the core `MaterialIcon` component body in `src/components/MaterialIcon/MaterialIcon.tsx`:
  - Import `useContext` from `react`; import `ThemeContext` from `src/theme/ThemeContext.ts` (NOT `useTheme()` — see research.md Decision 5)
  - Resolve `color`: `props.color ?? ctx?.colorScheme.onSurface ?? '#000000'`
  - When `ctx` is `null`, emit `__DEV__ && console.warn('[mui-native] MaterialIcon: no ThemeProvider found, falling back to #000000')`
  - In `__DEV__` mode, if `name` is not in `VALID_ICON_NAMES` (from `catalogue.ts`), emit `__DEV__ && console.warn(\`[mui-native] MaterialIcon: unknown icon name "${name}"\`)`
  - For `variant !== 'two-tone'`: pick the matching icon set from `iconSets.ts` and render `<IconSet name={name} size={size} color={resolvedColor} />`
  - Wrap entire component in `React.memo`
  - Default props: `variant = 'filled'`, `size = 24`
- [X] T009 [US1] Add Two Tone rendering branch in `src/components/MaterialIcon/MaterialIcon.tsx`:
  - Render a `<View style={{ position: 'relative', width: size, height: size }}>` wrapper
  - First child: `<TwoToneIcons name={name} size={size} color={resolvedColor} style={{ opacity: 1 }} />`
  - Second child: `<TwoToneIcons name={name} size={size} color={resolvedColor} style={[StyleSheet.absoluteFill, { opacity: 0.4 }]} />`

**Checkpoint**: `<MaterialIcon name="home" />` renders inside a `ThemeProvider`; `<MaterialIcon name="home" variant="two-tone" />` renders two overlapping glyphs.

## Phase 4: User Story 2 — `materialIconSource` Factory (Priority: P2)

**Goal**: `materialIconSource("star", "outlined")` returns an `IconSource`-compatible function usable with the existing `<Icon source={...} />` component.

**Independent Test**: Call `materialIconSource("star", "outlined")`, invoke the returned function with `{ size: 24, color: "#000" }`, assert the return value is a React element.

### Implementation

- [X] T010 [P] [US2] Write `src/components/MaterialIcon/materialIconSource.ts` *(depends only on Phase 2 — T004 + T007; does not require Phase 3 to be complete)*:
  - Signature: `export function materialIconSource(name: MaterialIconName, variant: IconVariant = 'filled'): IconSource`
  - The returned `IconSource` function receives `{ size, color }` — use these directly (no `useContext` call inside a non-component function)
  - Pick the matching icon set from `iconSets.ts` based on `variant`
  - Return `({ size, color }) => React.createElement(IconSet, { name, size, color })`
  - Import `IconSource` from `src/components/Icon/types.ts` (existing type — do not redefine)

**Checkpoint**: `<Icon source={materialIconSource("star", "outlined")} size={24} color="#000" />` renders the outlined star icon without errors.

## Phase 6: User Story 4 — Accessible Icon Labeling (Priority: P3)

**Goal**: Icons with `accessibilityLabel` are announced by screen readers; icons without it are hidden from the accessibility tree.

**Independent Test**: Render `<MaterialIcon name="home" accessibilityLabel="Navigate home" />` and assert `accessibilityRole === "image"` and the label is present on the element.

### Implementation

- [X] T013 [US4] Add accessibility props to `MaterialIcon` render output in `src/components/MaterialIcon/MaterialIcon.tsx`:
  - When `accessibilityLabel` is provided: set `accessible={true}`, `accessibilityRole="image"`, `accessibilityLabel={accessibilityLabel}` on the root element
  - When `accessibilityLabel` is absent: set `accessible={false}`, `accessibilityRole="none"` (mirrors existing `Icon` component behavior)
  - For Two Tone branch: apply the same props to the outer `<View>` wrapper

**Checkpoint**: `getByRole('image')` finds the element when `accessibilityLabel` is set; `queryByRole('image')` returns null when omitted.
